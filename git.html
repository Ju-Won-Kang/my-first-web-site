Git 무엇인가?

git은 형상 관리 도구(Configuration Management Tool) 중 하나로 분산형 관리 시스템이다.

git의 동작

working directory ->(ADD) Stage Area ->(COMMIT) Local Repository ->(PUSH) Remote Repository

git은 working directory에서 작업을 하고 Stage Area에 추가 후 모아서 Local Repository에 커밋 후 Remote Repository에 푸쉬를 한다.
커밋을 하게 되면 포인터인 "HEAD"는 main(master)를 가르키게 되고 main은 최신의 커밋id를 가르킨다.
여기서 디버깅을 위해서 이전 커밋 단계로 넘어가기 위해서 git checkout [git id] 를 사용해 "head"의 위치를 변경한다 .
변경한 뒤에는 git log를 통해 최근 까지의 커밋 id가 안나오는데 이럴 때 git log --all을 사용해 확인 후 git checkout [git id or main]로 다시 되돌리면 된다.
local repository에 commit할 때 git id가 생성되고 해당 git id를 가지고 checkout을 수행할 수 있다.
최신 상태로 돌리기 위해서는 #git checkout main 을 통해서 Branch를 main으로 설정한다. 이것을 확인하는 방법은 git graph 혹은 sourcetree에서 main이 있는 곳에 파랑색 태두리가 있으면 main Branch로 버전을 바꾼 것이다.

git log

git log의 옵션으로는
  • --all : git log의 전체를 보여줘라
  • --oneline : git log를 한줄로 보여줘라
  • --graph : graph로 분기 또는 연결된것을 시각적으로 보여줘라

    • HEAD & main(master)

  • HEAD : 현재의 working directory와 sStage Area가 어떤 버전이 만들어진 시점인지 가르키는 것
  • main(master) : Branch에서 작업한 마지막 버전이 무엇인지를 가르키는 것

    • Branch

    main은 로컬 저장소의 master, origin은 원격 저장소의 최신 버전

    충돌(conflict)

    협업자와 작업자가 같은 파일의 같은 행을 다른 내용을 수정하고 push할 때 협업자가 먼저 push 하고 작업자가 push하는 경우 reject 당해 pull을 하여 auto merge를 하려 했으나 안되게 하고 이것을 해결해달라고 하는 기능이다.
    특수 기호는 지우고 수정한 뒤 marge commit한 뒤 push 하면 된다.
